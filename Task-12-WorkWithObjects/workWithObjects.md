# Задача на работу с объектами

## [Реализация](./index.js)

## Постановка 
- Cоздайте объект, представляющий собой книгу. Объект должен иметь свойства, такие как: название книги, автор и год издания. Напишите методы для получения и изменения значений свойств книги.

## Реализованные функции
- Классовый подход (Book класс)
    - Плюсы:
        1. **Структурированность:** Использование класса создает явную структуру для объектов с методами.
        2. **Прототипирование:** Механизм прототипирования классов обеспечивает более эффективное использование памяти при создании множества объектов.
    - Минусы:
        1. **Больше кода:** Больше кода для создания класса и добавления методов к прототипу.
        2. **Может показаться избыточным:** Если вам не нужны преимущества, предоставляемые классами, использование классов может показаться избыточным.


- Объектный подход
    - Плюсы:
        1. **Простота:** Простой и лаконичный способ описания объекта с методами.
        2. **Гибкость:** Можно легко добавлять новые методы и свойства к объекту.
    - Минусы:
        1. **Неявность прототипирования:** Прототипирование менее явное в сравнении с классами.
        2. **Менее подходит для больших проектов:** Не так удобен при работе с большим количеством объектов с общими методами.
